Minimum Amplitude

Abstract

The Minimum amplitude (diParam.minAmplitude) parameter invalidates pixels where the amplitude (reflected light) drops below the minimum threshold.

Description

For each pixel, the amplitude value represents how much light was received by the imager. The minimum amplitude parameter provides a threshold that defines when the system should discard pixels with low amplitude. The images below show the amplitude image and the corresponding point cloud for a scene containing black totes that are made made out of a dark plastic and reflect very little light. As can be seen, part of the point cloud is missing where the amplitude of the pixel is below the chosen threshold value.

In certain cases, such as when black objects are in the field-of-view. Changing the default value from 20 to zero can be beneficial, because then more pixels are counted as valid pixels and the point cloud is more complete. Generally speaking, lowering the amplitude leads to more ambient noise and less accuracy in the distance measurement. In this case, we encourage you to test the filters available with the O3R to mitigate the noise from black objects measurements.

Note: black objects in the visible spectrum are not necessarily black in the near infrared range.

Note: The minimum amplitude threshold is applied to the non-normalized amplitude image. The numerical value of the normalized amplitude image might not correspond to expected values with the set threshold. The normalization factor used in our algorithm is accessible as part of the PCIC output and called ampNormalizationFactor.

Related topics

The minimum amplitude parameter is related to the maximum distance noise parameter: a low amplitude value with a high distance noise value ensures that more pixels are valid but will allow for a noisier measurement, requiring some filtering, for instance with the temporal filter.
