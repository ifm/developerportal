How to: Get 2D images¶

If you have an O3R-head with a 2D imager, you might want to get the 2D image and evaluate it. To do so, you need to physically connect the head (obviously). A head with two imagers, has two cables to connect to the VPU. One for each imager. Right now, all 2D imagers are also mapped to the TCP/IP ports 50020 and higher. At the same time, the 2D imager are only available if connected to (physical) port 0 and 1 on the VPU.

The general FrameGrabber() and ImageBuffer() do not need a specific configuration. They use the port number as a reference. It is however not possible to get amplitude data, etc. from a 2D frame. Again, fairly obvious.

Receive a 2D image¶

In [8]:
from ifmO3r.ifm3dTiny import Device, FrameGrabber, ImageBuffer

device = Device('192.168.0.69', 50021)
fg = FrameGrabber(device)
im = ImageBuffer()

fg.wait_for_frame(im, 5)
image_2d = im.image_2D()

print(image_2d[0:10])

Visualize the data¶

Using python and some standard packages, we can create a little viewer (note that the package ifmO3rViewer does not handle 2D images).

In [6]:
import matplotlib.pyplot as plt
from PIL import Image
import io
def showFrameMatplotLib(image_2d):
    inmemoryfile = io.BytesIO(image_2d)
    testJpeg = Image.open(inmemoryfile)
    im1 = plt.imshow(testJpeg)
    plt.show(block=True)
