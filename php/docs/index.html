Requirements

You will need the following things

  1. A working/live github repo that is publicacly accesible
  2. Either PHP server, or Github Actions running php
  3. Website to host the Doc API (GitHub Pages, or physical hosting both work)
  4. Web Browser for testing (eg: Google Chrome or Mozilla Firefox)

Be careful while editing the index.php file and the docapi.js file. If not edited properly, the design layout may break completely.
No support is provided for faulty customization.

Getting Started

The DocApi is a system used to pull your documentation from github, and display it on the developorportal website. It uses Javascript, PHP and Json to make it work, as well as github actions to add some automation in to the mix.

How to get it to work

First add this inside the head. This is the css file that will style the docapi menu. You can also add any markdown/readme styles you need. Or create a new css file and link it.

Second add this right before the </body>. This is used to parse the markdown from the github repo.

You will need to add this code below to your html page you want the docapi to show up. Be sure to change the "data-xxxxxx" info to match your repo.

  • data-repo: This is the username and repo from github. https://github.com/ifm/documentationtree/main/O3R (required)
  • data-branch: This is the branch the specific branch the documentation should be pulled from. https://github.com/ifm/documentation/tree/main/O3R (required/ Default: master)
  • data-subfolder: This is the base subfolder the documentation resideds in. https://github.com/ifm/documentation/tree/main/O3R (required)
  • data-file: This is the first table of contents README file that exisits. https://github.com/ifm/documentation/blob/main/O3R/README.md (required)

The only other thing you need to do is make sure the menu.json file the PHP created is in the correct spot. Default: /jquery/menu.json

Workflow

This section will explain the different parts to how this system works.

Html and Javascript

The html and javascript work together to give you the UI and the actual function of pulling the repo and display it on the page. The docapi.js file reads a menu.json file that is created by the php to get all the markdown files it needs to read, and displays them based on the order they were created in the Json file.

PHP and Github Action

The PHP works by parsing the markdown and looking for all of the table of contents documents in your documentation and building the menu.json file the docapi.js file needs to work. This menu.json file can be created manualy. But this is a way to partially automate the process. If you have the website hosted on github pages you will need to setup the github actions workflow to get this to work. See the "".github/workflows/main.yml" for more info.
